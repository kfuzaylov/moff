Getting started

Install with Bower

$ bower install moff --save

If you don't use bower you can download development and map or production version.

Setting up

Moff is initialized automatically on DOMContentLoaded event. But you can pass settings value for initialization by defining window.moffConfig object. If you do not define config object Moff it will be initialized with default setting values.

window.moffConfig = {
	// Website CSS breakpoints.
	// Default values from Twitter Bootstrap.
	// No need to set xs, because of Mobile first approach.
	breakpoints: {
		sm: 768,
		md: 992,
		lg: 1200
	},
	loadOnHover: true,
	cacheLiveTime: 2000
};

Initialization options

  • breakpoints - Sets break points which are used by framework to determine what is the current screen size.
    If you customized Twitter Bootstrap break points do not forget to change them for window.moffConfig too.
  • loadOnHover - Enables instant click effect. On mouse over it pre-loads data and stores in cache. And on click data will be loaded from cache.
    It works only for non mobile devices.
  • cacheLiveTime - Sets cache live time loaded on element hover. Default is 2 seconds. If you increase the value it starts to use more memory
    and may be the cause of showing old, expired data.

And include moff after config. 

<script src="bower_components/moff/dist/moff.min.js"></script>

Browser support

Moff supports major of modern web browsers.

  • Internet Explorer 9+
  • Mozilla Firefox
  • Google Chrome
  • Opera 12+
  • Apple Safari
